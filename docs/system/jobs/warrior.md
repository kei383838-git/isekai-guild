# 戦士

## 1. 基本方針

戦士は近接戦闘を得意とするジョブである。

高いステータスを持ち、
安定して戦える初心者向けジョブとして設計する。

## 2. 役割

* 前線で敵を受け止める
* 近接攻撃で安定してダメージを与える
* 危険な場面でも生き残りやすい

## 3. 特徴

* HPが高い
* 攻撃力が高い
* 防御力が高い
* 持てるアイテムが多い
* 近接戦闘に強い
* 遠距離攻撃や広範囲攻撃は苦手

## 4. 初期実装方針

初期試作では、戦士のみを実装対象とする。

まずは以下の要素を優先して形にする：

* 通常攻撃
* 基本ステータス差
* 戦士専用スキル1〜2個

## 5. 初期スキル

### 強撃

* 直線2マスの敵に攻撃する
* 通常攻撃よりやや高いダメージを与える
* SPを消費する

※高性能なため、SP消費でバランスを取る

### 回転斬り

* 自身の周囲8マスに攻撃する
* 威力は通常攻撃と同等
* SPを消費する

複数の敵に囲まれた際の対処スキルとして機能する

## 6. リソース

* スキル使用時はSPを消費する
* SPは今後、時間経過やアイテム等で回復する想定

## 7. 将来の発展候補

戦士系統の派生ジョブとして以下を想定する：

* 狂戦士
* 聖騎士
* 勇者
* 魔法剣士

そのため、戦士は近接前衛の基礎ジョブとして位置付ける。

# SP仕様

## 1. 基本方針

SPはスキル使用に必要なリソースである。

プレイヤーはSPを管理しながら、
通常攻撃とスキルを使い分けて戦う。

## 2. 基本値

* SPの初期値は50
* SPの最大値は100
* ダンジョン開始時は初期値で開始する

## 3. 使用方法

* スキル使用時にSPを消費する
* SPが不足している場合、そのスキルは使用できない

## 4. 回復方法

* 5ターンごとにSPを1回復する
* 拠点へ帰還するとSPは全回復する
* アイテムによってSPを回復できる

## 5. 初期スキル消費量

* 強撃：SP10
* 回転斬り：SP20

## 6. 設計意図

* 通常攻撃を基本としつつ、ここぞという場面でスキルを使う構造にする
* スキル連発を防ぎながら、使い惜しみしすぎないバランスにする
* 探索中の継戦能力に戦略性を持たせる
